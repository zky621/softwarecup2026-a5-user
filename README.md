# 中国软件杯 2026 A5 用户端

本仓库为中国软件杯 2026 A5 赛题「景区导览服务 AI 数字人」的用户端工程，面向游客侧 Web/H5 与微信小程序端。

项目基于 `uni-app + Vue 3 + TypeScript + Vite` 构建，目标是在移动端、小程序端和浏览器端提供统一的景区数字人导览入口，并与 Fay 数字人框架、景区知识库、路线规划和运营服务接口配合使用。

## 赛题背景

A5 赛题聚焦景区导览服务中的 AI 数字人应用。用户端承担游客进入景区后的第一交互界面，主要解决以下问题：

- 游客需要快速获得景点介绍、开放时间、游览建议和服务设施信息。
- 景区需要通过数字人降低人工咨询压力，并提供标准化、多轮、可追溯的导览服务。
- 多端入口需要统一体验，覆盖 Web/H5、微信小程序、扫码入口和现场屏幕联动场景。
- 导览服务需要与知识库、语音交互、数字人播报和后台运营系统形成闭环。

## 用户端定位

本仓库是 A5 方案中的 C 端用户入口，负责承载游客直接使用的导览功能：

- 数字人导览首页
- 景区知识问答
- 游览路线推荐
- 语音对话入口
- 推荐点位与当前行程展示
- 游客登录、票务预约和导览偏好
- Web/H5 与微信小程序双端构建

## Fay 原有能力

本项目延续 Fay 数字人框架的原有定位：面向终端应用的开源数字人框架。Fay 向上适配数字人模型与大语言模型，向下接入 TTS、ASR、知识库和控制接口，可用于网站、小程序、App、大屏和第三方业务系统。

Fay 原有能力包括：

- 支持全离线使用
- 支持全时流式交互
- 支持 OpenAI 兼容接口的大语言模型
- 支持自定义 ASR、TTS 和数字人模型
- 支持文字交互、语音交互和数字人驱动接口
- 支持自动播报模式
- 支持自定义知识库、自定义问答对和自定义人设信息
- 支持唤醒与打断对话
- 支持服务端和单机模式
- 支持多用户、多路并发
- 支持 MCP 工具管理
- 支持配置管理中心

用户端并不替代 Fay 主工程，而是作为游客侧入口与 Fay 后端、管理端和数字人运行端配合使用。

## 技术栈

```text
跨端框架：uni-app
前端框架：Vue 3
开发语言：TypeScript
构建工具：Vite
包管理器：pnpm
目标端：Web/H5、微信小程序
```

## 页面结构

```text
src/pages/index/index.vue   导览首页：数字人状态、实时交互、路线、点位和行程
src/pages/about/about.vue   服务能力：问答、路线规划、语音讲解、附近服务
src/pages/me/me.vue         我的：游客资料、票务预约、偏好设置和登录入口
src/tabbar/                 用户端底部导航
env/.env                    应用标题、端口、后端地址和小程序配置
```

## 本地运行

```bash
pnpm install --frozen-lockfile
pnpm dev:h5 --host 127.0.0.1
```

默认访问地址：

```text
http://127.0.0.1:9000/
```

## 构建

```bash
pnpm type-check
pnpm build:h5
pnpm build:mp-weixin
```

构建产物：

```text
dist/build/h5          Web/H5 产物
dist/build/mp-weixin   微信小程序产物
```

如果本机未配置微信开发者工具 CLI，微信小程序构建仍会生成 `dist/build/mp-weixin`，但不会自动打开开发者工具。可手动打开微信开发者工具并导入该目录。

## 与 Fay 主工程关系

推荐工程分工：

```text
Software-Digital-human      Fay 主工程、管理端、数字人运行端、接口服务
softwarecup2026-a5-user     游客侧 uni-app 用户端
```

用户端后续接入 Fay 主工程时，应优先对接：

- 数字人对话接口
- 景区知识库问答接口
- 语音识别与语音播报接口
- 景点、路线、票务和服务设施数据接口
- 登录、用户偏好和问答历史接口

## 赛题交付建议

本用户端适合作为 A5 方案中的游客侧演示入口，与管理端、数字人运行端、知识库和运营看板共同组成完整交付：

- 游客扫码或打开 H5 进入导览首页。
- 通过文字或语音向 Fay 提问。
- Fay 结合景区知识库返回讲解、路线和服务建议。
- 用户端展示推荐点位、当前行程和游客偏好。
- 管理端维护知识、点位、数字人配置和运营数据。
